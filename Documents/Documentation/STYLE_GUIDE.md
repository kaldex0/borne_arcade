# Style guide documentation

## Principes

- Docs-as-Code: la doc est versionnee comme le code.
- L'IA propose des patchs, jamais des modifications automatiques non revues.
- Toute modification de code implique une mise a jour de doc associee.

## Ton et format

- Francais clair, phrases courtes.
- Titres en phrase, pas tout en majuscules.
- Utiliser des listes courtes et ordonnees.
- Les commandes doivent etre dans des blocs de code.

## Structure

- Une page = un sujet.
- Mettre les pre-requis avant les etapes.
- Terminer par une section "Verification" si possible.

## Patterns recommandes

- How-to: objectif, prerequis, etapes, verification, depannage.
- Reference: definitions courtes, liens.

## IA

- Toujours fournir un diff et un resume.
- Ne pas ecraser la doc sans validation humaine.
